Section 02 Deploy Website to Vercel

Vercel provides automatic deployments from your Git repository. Every push to the main branch triggers a new production build.

  1. Sign in to Vercel
    Go to vercel.com and sign in using your GitHub account (recommended) or email.
  2. Import your repository
    Click "Add New…" → "Project" from the dashboard. Select your GitHub repository from the list. If it doesn't appear, click "Adjust GitHub App Permissions" to grant Vercel access.
  3. Configure project settings
    • Framework Preset: Vercel auto-detects (e.g., Next.js, React, Vite). Verify it's correct.
    • Root Directory: Leave as ./ unless your code is in a subdirectory.
    • Build Command: Usually auto-detected. For Next.js: next build.
    • Output Directory: Leave default unless using a custom setup.
    • Environment Variables: Add any required env vars (API keys, etc.). Click "Add" for each one.
  4. Click "Deploy"
    Vercel will clone the repo, install dependencies, run the build, and deploy. Wait for the build to complete (usually 1–3 minutes).
  5. Verify the deployment
    Once complete, Vercel gives you a preview URL like your-project.vercel.app. Open it and verify the website works correctly.
Tip: Automatic Deployments Every future push to your main branch will automatically redeploy. Preview deployments are created for pull requests on other branches.
Common Issue: Build Fails If the build fails, check the build logs in the Vercel dashboard. Common causes: missing environment variables, incorrect Node.js version (set in package.json engines field), or unresolved imports.
Section 03 Connect Custom Domain

This section covers connecting your custom domain (purchased from GoDaddy, Namecheap, or any registrar) to your Vercel deployment.

Step A — Add Domain in Vercel

  1. Open your project in Vercel
    Go to your project dashboard → Settings → Domains.
  2. Enter your domain
    Type your domain (e.g., yourdomain.com) and click "Add".
  3. Choose redirect behavior
    Vercel will ask how to handle www vs non-www:
    • Recommended: Use www.yourdomain.com as primary, redirect bare domain to it.
    • OR: Use bare domain as primary, redirect www to it.
    Add both versions. Vercel will show the DNS records you need to configure.

Step B — Configure DNS at Your Registrar

Vercel will show you the exact DNS records needed. Here is what you'll typically add:

# For bare domain (yourdomain.com):
Type: A   Name: @   Value: 76.76.21.21   TTL: 600

# For www subdomain:
Type: CNAME   Name: www   Value: cname.vercel-dns.com   TTL: 600

Configuring DNS on GoDaddy

  1. Open GoDaddy DNS Management
    Log in to godaddy.com → My Products → Find your domain → DNS (or "Manage DNS").
  2. Delete existing A records pointing elsewhere
    If there's an existing A record for @ pointing to a "Parked" page, delete it or edit it.
  3. Add the A record
    Click "Add Record" → Type: A, Name: @, Value: 76.76.21.21, TTL: 600 (or 1/2 Hour). Click Save.
  4. Add/Edit the CNAME record
    Add or edit a CNAME record → Name: www, Value: cname.vercel-dns.com, TTL: 600. Click Save.
  5. Wait for DNS propagation
    Changes take 5 minutes to 48 hours to propagate globally. Usually under 30 minutes.
For Namecheap Users Go to Domain List → Manage → Advanced DNS → Add the same records. For the CNAME, use www as Host (not the full domain).
Critical: Do NOT Change Nameservers Unless Instructed If you only need to point your domain to Vercel, add the A and CNAME records above. Do NOT change your nameservers to Vercel's nameservers unless you want Vercel to manage all your DNS (including email MX records). Changing nameservers will break any existing email setup.
Section 04 Verify SSL & Domain Configuration

After DNS propagation, verify everything is working correctly.

  1. Check domain status in Vercel
    Go to your project → Settings → Domains. Both domains should show a green checkmark and say "Valid Configuration". If it shows an error, DNS hasn't propagated yet — wait and retry.
  2. Verify SSL certificate
    Vercel automatically generates a free SSL certificate via Let's Encrypt. Visit https://yourdomain.com and confirm the padlock icon appears in the browser address bar. This happens automatically after DNS propagates.
  3. Test redirects
    Verify these all work correctly:
    • http://yourdomain.com → redirects to https://www.yourdomain.com
    • http://www.yourdomain.com → redirects to https://www.yourdomain.com
    • https://yourdomain.com → redirects to https://www.yourdomain.com
    All four variations must land on the same final URL (your primary domain with HTTPS).
  4. Verify DNS propagation globally
    Use dnschecker.org to confirm your A and CNAME records have propagated worldwide.
Redirect Type Matters Vercel should use 308 (Permanent Redirect) for domain redirects. Check in Settings → Domains that the redirect type says "308" not "307". A 307 is temporary and won't pass SEO value. If it shows 307, remove and re-add the domain.
Section 05 Google Search Console Setup

Google Search Console (GSC) lets you monitor how Google sees your website, submit sitemaps, check indexing status, and diagnose SEO issues. This is the most critical tool for search visibility.

Official URL: search.google.com/search-console

Step A — Add Property

  1. Sign in to Google Search Console
    Use the same Google account you'll use for Analytics and Business Profile.
  2. Click "Add Property"
    You'll see two options:
    • Domain (Recommended): Covers ALL variations (www, non-www, http, https, subdomains) in one property. Requires DNS verification only.
    • URL Prefix: Tracks only the exact URL entered. Supports multiple verification methods. Use this if you cannot access DNS settings.
  3. Choose "Domain" and enter your domain
    Enter yourdomain.com (without http/https/www). Click Continue.

Step B — Verify Ownership

  1. Copy the TXT record provided by Google
    Google will display a TXT record like:
    google-site-verification=xxxxxxxxxxxxxxxxxxxxxx
  2. Add the TXT record to your DNS
    Go to your domain registrar's DNS settings (GoDaddy, Namecheap, etc.):
    • Type: TXT
    • Name/Host: @
    • Value: The full google-site-verification=... string
    • TTL: 600 (or default)
  3. Click "Verify" in Google Search Console
    Wait a few minutes for DNS to propagate, then click Verify. If it fails, wait 15–30 minutes and try again.
Alternative: HTML File Verification (URL Prefix only) Download the HTML file from Google, place it in your project's public/ folder (e.g., public/googleXXXXXXXX.html), deploy to Vercel, then verify. Google will check https://yourdomain.com/googleXXXXXXXX.html.

Step C — Submit Sitemap

  1. Go to "Sitemaps" in the left sidebar
    In Google Search Console, click Sitemaps under the Indexing section.
  2. Enter your sitemap URL
    Type sitemap.xml in the field and click "Submit".
    Full URL will be: https://www.yourdomain.com/sitemap.xml
  3. Verify status shows "Success"
    After Google processes it, the status should say "Success" with the number of discovered URLs.

Step D — Request Indexing

  1. Use the URL Inspection tool
    In the top search bar of GSC, paste your homepage URL: https://www.yourdomain.com
  2. Click "Request Indexing"
    If the page is not yet indexed, click "Request Indexing". Google will prioritize crawling this page. Repeat for your most important pages (services, about, contact).
Indexing Takes Time Google typically indexes a new site within 4–28 days. You cannot speed this up. Submitting the sitemap and requesting indexing are the only actions you can take. Do NOT submit the same URL repeatedly.

Step E — Recommended GSC Settings

  1. Set preferred country targeting — Settings → International Targeting (if applicable).
  2. Add users — Settings → Users and Permissions → Add team members with appropriate access levels (Owner / Full / Restricted).
  3. Check Coverage report — Pages → check for any errors or warnings.
  4. Monitor Core Web Vitals — Experience → Core Web Vitals → review any issues flagged.
Section 06 Google Analytics (GA4) Setup

Google Analytics 4 tracks visitor behavior, traffic sources, conversions, and more. It provides data you need to understand how users interact with your website.

Official URL: analytics.google.com

Step A — Create Account & Property

  1. Go to Google Analytics and sign in
    Use the same Google account as Search Console.
  2. Click "Start measuring" (new) or Admin → Create → Property
    Fill in:
    • Account Name: Your company name (e.g., "Worksthal")
    • Property Name: Your website name (e.g., "Worksthal Website")
    • Reporting Time Zone: Your local timezone
    • Currency: Your primary business currency
  3. Select business details
    Choose your industry category, business size, and objectives (generate leads, drive sales, etc.).
  4. Accept Terms of Service
    Review and accept the Google Analytics Terms of Service for your country.

Step B — Set Up Data Stream

  1. Choose "Web" as platform
    After creating the property, select Web as your data stream type.
  2. Enter your website URL
    Enter https://www.yourdomain.com and give the stream a name (e.g., "Main Website").
  3. Enable Enhanced Measurement
    Toggle this ON to automatically track page views, scrolls, outbound clicks, site search, video engagement, and file downloads.
  4. Copy your Measurement ID
    You'll receive a Measurement ID that looks like G-XXXXXXXXXX. Copy this — you'll need it for installation.

Step C — Install the GA4 Tag on Your Website

Add the Google tag to your website code. For Next.js, add it in your root layout file (app/layout.tsx).

<!-- Place inside <head> of your layout -->
<Script
  src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"
  strategy="afterInteractive"
/>
<Script strategy="afterInteractive">
  {`window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX');`}
</Script>
Do NOT Add the Tag Twice Only add one Google tag per page. If using Google Tag Manager, install GTM instead of the gtag.js snippet — never use both simultaneously.

Step D — Verify Installation & Configure Settings

  1. Verify with Realtime report
    In GA4, go to Reports → Realtime. Open your website in another tab. You should see yourself as an active user within 30 seconds.
  2. Change data retention to 14 months
    Admin → Data Settings → Data Retention → Change from 2 months to 14 months. Click Save. (This is frequently missed!)
  3. Link Google Search Console
    Admin → Product Links → Search Console Links → Link. Select your Search Console property. This imports search query data into GA4.
  4. Set up Key Events (Conversions)
    Admin → Events → mark important events as "Key Events" (form submissions, button clicks, page views on contact page, etc.).
  5. Exclude internal traffic
    Admin → Data Streams → your stream → Configure tag settings → Define Internal Traffic → add your office/home IP addresses so your own visits don't skew data.
Link Google Ads If running Google Ads, link it immediately: Admin → Product Links → Google Ads Links. This enables conversion tracking and audience sharing between the platforms.
Section 07 Google Business Profile Setup

Google Business Profile (GBP) makes your business appear in Google Maps and the local knowledge panel in search results. Even online-only businesses benefit from having one.

Official URL: business.google.com

Step A — Create Your Business Profile

  1. Go to business.google.com and sign in
    Use the same Google account. Click "Manage now" or "Add your business".
  2. Enter your business name
    Type your exact business name. If it doesn't appear in suggestions, select "Add your business to Google".
  3. Choose your business category
    Select the most accurate primary category. Examples:
    • IT/Tech: "Software Company", "Internet Marketing Service", "Web Designer"
    • Agency: "Marketing Agency", "Advertising Agency"
    • Consulting: "Business Management Consultant", "IT Consultant"
    You can add additional categories later.
  4. Set your location type
    • Physical location: If customers visit your office.
    • Service area business: If you go to customers or work remotely. Enter the cities/regions you serve (up to 20).
  5. Add contact information
    Enter your phone number and website URL (https://www.yourdomain.com).

Step B — Verify Your Business

  1. Google will assign a verification method
    You cannot choose — Google determines the method based on your business type:
    • Video verification (most common for new businesses): Record a 30+ second video showing your business location, signage, and proof of management.
    • Phone/SMS: Receive a code via phone call or text.
    • Email: Receive a code via email.
    • Postcard: Physical postcard mailed to your business address (5–14 days).
  2. Complete verification
    Follow the prompts for your assigned method. Enter the code when received. Verification review can take up to 7 business days.

Step C — Optimize Your Profile

  1. Add business hours — Even for online businesses, set hours when customers can reach you.
  2. Write a business description — Up to 750 characters. Include your key services and value proposition. Use relevant keywords naturally.
  3. Upload photos — Add a logo, cover photo, and 5+ photos of your team/office/work. Businesses with photos receive 42% more direction requests.
  4. Add services/products — List your specific services with descriptions and pricing (if applicable).
  5. Set up messaging — Enable the messaging feature so customers can contact you directly from Google.
  6. Create your first post — Write a "What's New" post about your business. Post regularly (weekly is ideal).
NAP Consistency Your Name, Address, and Phone number must be exactly identical across your website, Google Business Profile, and all directory listings. Even small differences (e.g., "St." vs "Street") can hurt local SEO.
Section 08 Bing Webmaster Tools Setup

Bing powers both Bing and Yahoo search results (~6% of global search traffic combined). Setup takes under 10 minutes if you've already set up Google Search Console.

Official URL: bing.com/webmasters

  1. Sign in to Bing Webmaster Tools
    Go to bing.com/webmasters. Sign in with a Microsoft, Google, or Facebook account.
  2. Import from Google Search Console (fastest method)
    Bing offers a one-click import from GSC. Click "Import from GSC", sign in with your Google account, and select your property. This copies your sitemap and settings automatically.
  3. Alternative: Manual setup
    If not importing, click "Add your site manually" and enter your URL. Verify ownership via:
    • CNAME record (recommended) — Add a CNAME record to your DNS
    • XML file — Upload a file to your site's root
    • Meta tag — Add a meta tag to your homepage
  4. Submit your sitemap
    If you didn't import from GSC, go to Sitemaps in the left menu, enter https://www.yourdomain.com/sitemap.xml, and click Submit.
  5. Submit URLs for indexing
    Go to URL Submission and submit your key pages for faster indexing.
Quick Win The "Import from GSC" method takes under 2 minutes and carries over your verified ownership, sitemaps, and geo-targeting settings.

Section 10 Post-Launch Verification Checklist

Use this checklist to verify everything is properly set up. Go through each item after completing all the sections above.

Hosting & Domain

  • Website loads at https://www.yourdomain.com
  • SSL padlock icon visible in browser
  • All URL variations redirect correctly (http → https, non-www → www)
  • Redirect type is 308 (permanent), not 307 (temporary)
  • Vercel dashboard shows "Valid Configuration" for all domains
  • DNS propagation confirmed on dnschecker.org

Technical SEO

  • robots.txt is accessible at /robots.txt and allows crawling
  • sitemap.xml is accessible at /sitemap.xml and lists all pages
  • All pages have unique <title> tags (50-60 characters)
  • All pages have unique <meta name="description"> tags (150-160 characters)
  • Open Graph tags are present for social media sharing
  • Favicon is set and displays in browser tab
  • No broken links (test with brokenlinkcheck.com)

Google Search Console

  • Property verified (green checkmark)
  • Sitemap submitted and status shows "Success"
  • Homepage inspected and indexing requested
  • No critical errors in Coverage/Pages report
  • Core Web Vitals report reviewed

Google Analytics (GA4)

  • Tracking code installed on all pages
  • Realtime report shows active users when visiting site
  • Data retention set to 14 months
  • Search Console linked to GA4
  • Internal traffic filter configured
  • Key Events (conversions) defined
  • Google Ads linked (if running ads)

Google Business Profile

  • Business profile created and verification submitted
  • Business name, address, phone match website exactly
  • Category selected accurately
  • Business description written (up to 750 characters)
  • Logo and cover photo uploaded
  • Website URL added and correct
  • Business hours set
  • First post created

Bing & Other Listings

  • Bing Webmaster Tools verified and sitemap submitted
  • LinkedIn Company Page created with website link
  • Facebook Business Page created with website link
  • At least 2 industry directories submitted (e.g., Clutch, GoodFirms)
  • NAP (Name, Address, Phone) is consistent across all listings
